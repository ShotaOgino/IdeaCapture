# タスク完了時のチェックリスト

## コード変更後の確認事項

### 1. ビルドの確認
- [ ] プロジェクトがエラーなくビルドできる
- [ ] 警告が発生していないか確認

### 2. テスト
- [ ] 既存のユニットテストが通る
- [ ] 新機能に対するテストを追加（必要に応じて）

### 3. コード品質
- [ ] Swiftの命名規約に従っている
- [ ] 適切なアクセス修飾子を使用
- [ ] メモリリークの可能性がないか確認
- [ ] 非同期処理が適切にMainActorで実行される

### 4. Git操作
- [ ] 変更内容を確認 (`git diff`)
- [ ] 不要なファイルがコミットされないか確認
- [ ] 適切なコミットメッセージを記述

### 5. ドキュメント
- [ ] 重要な変更はコメントで説明
- [ ] 複雑なロジックには説明コメントを追加

## 注意事項
- Xcodeプロジェクトファイル (`.pbxproj`) の変更は慎重に
- UserDefaultsのキー名は一貫性を保つ
- 音声認識の権限チェックを忘れない
- Widget Extensionとの共有データは App Groups を使用
